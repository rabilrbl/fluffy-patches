## Root Cause Analysis

The `ExceptionInInitializerError` in `BaseActivity.onCreate` indicates:

1. **The native library does MORE than populate strings** - It patches Kotlin runtime metadata at a deep level
2. **Simply initializing strings is insufficient** - The library likely:
   - Patches Kotlin class metadata
   - Modifies runtime type information
   - Initializes critical data structures that Kotlin reflection depends on

3. **ClassReference NPE** - The Kotlin `ClassReference` class fails because the metadata it expects hasn't been properly initialized by the native library.

## Conclusion

For x86_64 AVD, we cannot bypass pairip with smali patching alone because:
- The native library performs runtime Kotlin metadata patching
- This patching cannot be reproduced via smali code modifications
- A proper mock native library would be needed

For ARM64 physical device:
- The original library loads but fails its own integrity check
- Results in `length_error was thrown in -fno-exceptions mode with message "vector"`
